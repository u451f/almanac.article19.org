Oblivious HTTP Application Intermediation | OHAI

OHAI aims to define a protocol to anonymise HTTP requests using a partly-trusted intermediary. HTTP requests can reveal information about the client to a server. Over time, servers are able to use this information to assemble profiles of the clientâ€™s behaviour and make inferences about the people that use these clients. One widely-anticipated use of OHAI is a way to mask DNS-over-HTTPS (DoH) queries, making it difficult for anyone to aggregate information about specific DNS requests made from a given client. This can provide a degree of protection against servers correlating requests and increase user privacy.

  • Standad under development: Oblivious HTTP sets out a system to forward encrypted HTTP messages from a client to an origin server without the server being able to link the requests back to the client.
